# **Задание 2 — Очередь**

## **Цель**
Добавить N элементов в очередь, затем извлечь первые 5 элементов.

## **Структура данных**
Класс `Queue` содержит:

- `enqueue(int)` — добавление в конец  
- `dequeue()` — удаление из начала  
- `print()` — вывод  

Узел (`Node`) содержит:

- `data`  
- `next`  

## **Алгоритм работы**
### enqueue:
- Если очередь пустая → `head = tail = new_node`  
- Иначе → `tail->next = new_node`, `tail = new_node`

### dequeue:
- Удаляется `head`  
- `head = head->next`  
- Если очередь стала пустой → `tail = nullptr`

## **Защита от дурака**
- Проверка корректности ввода  
- Проверка на пустую очередь  
- tail корректно обнуляется  
- Безопасная функция `safe_input()`  

## **Соответствие требованиям**
- Очередь реализована вручную  
- Память освобождается  
- Адреса head и tail выводятся  
- FIX_ME‑комментарии встроены  

## **Основные исправления (FIX_ME)**
- Переименование структур и методов  
- tail обнуляется при опустошении  
- Добавлена безопасная функция ввода

# **Задание 3 — Односвязный список**

## **Цель**
Вставить значение **M** перед каждым вторым элементом односвязного списка и вывести указатель на последний элемент.

## **Структура данных**
Реализован класс `LinkedList`, содержащий:

- `add_node(int)` — добавление в конец списка  
- `insert_before_every_second(int)` — вставка M перед каждым вторым элементом  
- `print_list()` — вывод списка  
- `get_last_node()` — получение указателя на последний узел  
- `clear_list()` — освобождение памяти  

Узел списка (`Node`) содержит:

- `data`  
- `next`  

## **Как работают указатели**
Вставка перед вторым элементом требует **двух указателей**:

- `cur` — текущий узел  
- `prev` — предыдущий узел  

Потому что в односвязном списке **нет ссылки назад**, и чтобы вставить узел *перед* текущим, нужно знать, кто стоит перед ним.

Алгоритм:

1. Идём по списку: `prev → cur`.  
2. Считаем элементы: `count++`.  
3. Если `count % 2 == 0`, значит это второй, четвёртый, шестой элемент.  
4. Вставляем новый узел M между `prev` и `cur`.  
5. Продолжаем обход.

## **Алгоритм вставки**
```
prev->next = new_node
new_node->next = cur
```

Если вставка перед самым первым элементом — обновляем `head_`.

## **Защита от дурака**
- Проверка корректности ввода количества элементов  
- Проверка корректности ввода каждого значения  
- Проверка, что список содержит минимум 2 элемента  
- Проверка на пустой список  
- Безопасная функция `safe_input()`  

## **Соответствие требованиям**
- Односвязный список реализован вручную  
- Память освобождается  
- Вставка корректна  
- Указатель на последний элемент выводится  
- Дружественный интерфейс  
- FIX_ME‑комментарии встроены  

## **Основные исправления (FIX_ME)**
- Переименование методов  
- Исправлена логика вставки  
- Добавлена безопасная функция ввода  
- Исправлены вызовы старых методов

# **Задание 4 — Двусвязный список (между min и max)**

## **Цель**
Вывести элементы, находящиеся **между минимальным и максимальным значениями** списка.

## **Структура данных**
Класс `DoublyList` содержит:

- `push_back(int)`  
- `print_list()`  
- `find_min_max(Node*&, Node*&)`  
- `is_first_before_second(Node*, Node*)`  
- `print_between_min_max()`  
- `clear()`  

Узел (`Node`) содержит:

- `data`  
- `prev`  
- `next`  

## **Алгоритм**
1. Находим min и max.  
2. Определяем, кто встречается раньше.  
3. Идём от первого к второму, выводя элементы между ними.  
4. Если между ними нет элементов — выводим сообщение.  

## **Как работают указатели**
- `prev` и `next` позволяют двигаться в обе стороны  
- `find_min_max` проходит весь список  
- `is_first_before_second` определяет порядок узлов  
- `print_between_min_max` выводит элементы между ними  

## **Защита от дурака**
- Проверка корректности ввода  
- Проверка на пустой список  
- Проверка, что min и max существуют  
- Проверка, что между ними есть элементы  
- Безопасная функция `safe_input_int()`  

## **Соответствие требованиям**
- Двусвязный список реализован вручную  
- Память освобождается  
- Логика min/max корректна  
- FIX_ME‑комментарии встроены  

## **Основные исправления (FIX_ME)**
- Переименование структур и методов  
- Исправлена логика определения порядка  
- Добавлена безопасная функция ввода

# **Задание 5 — STL‑контейнеры (дек + список)**

## **Цель**
Вставить **последние 5 элементов дека** в **обратном порядке**  
**перед пятым с конца элементом списка**, используя **один insert**.

## **Структура данных**
Используются:

- `deque<int>`  
- `list<int>`  

Функции:

- `input_container()` — заполнение (клавиатура / рандом / файл)  
- `print_container()` — вывод  
- `insert_last_five_reversed()` — вставка  

## **Алгоритм**
1. Проверяем, что оба контейнера имеют ≥ 5 элементов.  
2. Находим итератор на пятый с конца элемент списка.  
3. Берём последние 5 элементов дека.  
4. Разворачиваем их.  
5. Одним вызовом `insert()` вставляем в список.  

## **Защита от дурака**

- Проверка открытия файла  
- Проверка размеров контейнеров  
- Безопасная функция `safe_input_int()`  

## **Соответствие требованиям**
- Используются только STL‑контейнеры  
- Вставка одним insert  
- Дружественный интерфейс  
- FIX_ME‑комментарии встроены
