

#  **Задание 1 — Односвязный список**

##  Цель  
Вставить значение **M** перед каждым вторым элементом односвязного списка.  
Вывести указатель на последний элемент.

##  Структура данных  
Реализован класс `LinkedList`, содержащий:

- `add_node(int)` — добавление в конец  
- `insert_before_every_second(int)` — вставка M  
- `print_list()` — вывод  
- `get_last_node()` — получение последнего узла  
- `clear_list()` — освобождение памяти  

Узел списка представлен структурой `Node`.

##  Алгоритм вставки  
1. Идём по списку.  
2. Счётчик `count` увеличивается на каждом шаге.  
3. Если `count % 2 == 0`, вставляем новый узел перед текущим вторым элементом.  
4. Продолжаем обход.

##  Защита от дурака  
- Проверка корректности ввода количества элементов.  
- Проверка корректности ввода каждого значения.  
- Проверка, что список содержит минимум 2 элемента перед вставкой.  
- Проверка на пустой список при выводе.  
- Безопасная функция `safe_input()` предотвращает краш при вводе текста вместо числа.

##  Соответствие требованиям  
- Односвязный список реализован вручную  
- Память освобождается  
- Вставка корректна  
- Указатель на последний элемент выводится  
- Дружественный интерфейс  
- Ввод с клавиатуры

#  **Задание 2 — Очередь (TQueue)**

##  Цель  
Добавить N элементов в очередь, затем извлечь 5 первых элементов с помощью `dequeue`.

##  Структура данных  
Реализована структура:

- `Node` — элемент очереди  
- `TQueue` — содержит `head` и `tail`  

Методы:

- `enqueue(TQueue&, int)`  
- `dequeue(TQueue&)`  

##  Алгоритм работы  
### enqueue:
- Если очередь пустая → head = tail = новый узел  
- Иначе tail->next = новый узел  

### dequeue:
- Удаляется head  
- head = head->next  
- Если очередь стала пустой → tail = nullptr  

##  Защита от дурака  
- Проверка корректности ввода количества элементов.  
- Проверка корректности ввода каждого значения.  
- Проверка на пустую очередь перед удалением.  
- tail корректно обнуляется.  
- Безопасная функция `safe_input()`.

##  Соответствие требованиям  
- Очередь реализована вручную  
- Память освобождается  
- Извлекаются 5 элементов  
- Адреса head и tail выводятся  
- Дружественный интерфейс  
- Ввод с клавиатуры  
- FIX_ME‑комментарии встроены  
- Стиль Google соблюдён

# **Задание 4 — Двусвязный список (между min и max)**

## Цель  
Дан двусвязный список с различными значениями.  
Нужно вывести **значения элементов, находящихся между минимальным и максимальным элементами**, в том порядке, как они расположены в списке.

## Структура данных  
Реализован класс `DoublyList`, содержащий:

- `push_back(int)` — добавление элемента в конец  
- `print_list()` — вывод списка  
- `find_min_max(Node*&, Node*&)` — поиск указателей на минимальный и максимальный элементы  
- `is_first_before_second(Node*, Node*)` — проверка, кто встречается раньше в списке  
- `print_between_min_max()` — вывод элементов между min и max  
- `clear()` — очистка списка и освобождение памяти  

Узел списка представлен структурой `Node` с полями:

- `int data`  
- `Node* prev`  
- `Node* next`  

## Алгоритм вывода между min и max  
1. В `find_min_max` проходим по списку и находим узлы с минимальным и максимальным значениями.  
2. В `is_first_before_second` определяем, кто идёт раньше — min или max.  
3. В `print_between_min_max` выбираем `start` и `end` в зависимости от порядка.  
4. Идём от `start->next` до `end`, выводя все значения.  
5. Если между ними нет элементов — выводим сообщение, что элементы отсутствуют.

## Защита от дурака  
- Проверка корректности ввода количества элементов.  
- Проверка корректности ввода каждого значения через `safe_input_int()`.  
- Проверка на пустой список в `find_min_max()` и `print_between_min_max()`.  
- Проверка случая, когда min и max совпадают.  
- Сообщение, если между min и max нет элементов.  

## Соответствие требованиям  
- Двусвязный список реализован вручную, без STL.  
- Стандартный набор методов: добавление, вывод, очистка.  
- Используются процедуры/методы для поиска и вывода.  
- Память корректно освобождается в `clear()` и деструкторе.  
- Ввод с клавиатуры, дружественный интерфейс.  
- Все имена приведены к стилю (английские идентификаторы, CamelCase для типов, snake_case для функций при необходимости).  
- FIX_ME‑комментарии встроены и показывают, что было исправлено.
